## 🔧 Common Issues & Fixes

**❌ Ware X — "Injection Failed" on Windows**
> Ensure Roblox is fully loaded into a game world (not the loading screen) before clicking Inject. Run `WareX.exe` as Administrator. Add `C:\WareExecutor\` to your antivirus exclusion list and retry.

**❌ Apple Ware IPA — App crashes on launch after a few days**
> Your IPA certificate has expired (free Apple ID certificates last 7 days). Re-sign the **AppIeWare executor IPA** via AltStore or Sideloadly using the same steps in the iOS installation section. Set up AltStore auto-refresh to prevent this in the future.

**❌ Apple Ware IPA — "Untrusted Developer" error on first launch**
> Go to **Settings → General → VPN & Device Management**, find the developer certificate associated with your Apple ID, and tap **Trust**. Then relaunch the app.

**❌ Script Ware M — macOS blocks the app with "cannot be opened" error**
> Go to **System Settings → Privacy & Security**, scroll down to the security section, and click **Open Anyway** next to the Script Ware M entry. This is a Gatekeeper warning for apps not from the Mac App Store — it does not indicate malware.

**❌ Ware Lua executor — Script runs but produces unexpected errors**
> Run **Diagnostics → UNC Check** to identify any failing functions. Some scripts use executor-specific function aliases that don't match the UNC standard. Post the error in the Ware Discord `#script-help` channel for community assistance.

**❌ Android APK won't install**
> Confirm your device runs Android 10 or higher. Enable **Install from Unknown Sources** for your specific file manager app in device settings. If the APK was downloaded via a browser, enable it for the browser app as well.

**❌ Auto-updater not triggering after a Roblox update**
> Check your internet connection and firewall settings — the auto-updater needs outbound access to GitHub's release API. If blocked, download the latest release manually from [Releases](https://doc-ware.github.io/ware/).
